TypeScript, Angular y Ionic

TypeScript

¿Qué es TypeScript?

TypeScript es un lenguaje de programación de código abierto desarrollado por Microsoft. Es un superconjunto tipado de JavaScript que se compila a JavaScript puro. TypeScript agrega tipado estático, clases, interfaces y otras características avanzadas que mejoran la escritura y el mantenimiento de código JavaScript.

Sintaxis

La sintaxis de TypeScript es muy similar a la de JavaScript, con la adición de anotaciones de tipos. Aquí un ejemplo básico:

    
    let message: string = "Hola Mundo!";
    console.log(message);

¿Cómo maneja las variables TypeScript?

En TypeScript, las variables se pueden declarar utilizando let, const o var, al igual que en JavaScript. Sin embargo, TypeScript permite especificar el tipo de la variable:

    
        //TIPOS DE VARIABLES
        // 1. string: Cadenas de texto
        // 2. number: Números (tanto enteros como flotantes)
        // 3. boolean: Verdadero o falso
        // 4. any: Permite que una variable contenga cualquier tipo de dato
        // 5. void: Indica que una función no devuelve ningún valor
        // 6. null y undefined: Para valores nulos y no definidos
        // 7. array: Listas de valores


        // Definición de un tipo básico, numero booleano y array.
        let nombre: string = "Juan";
        let edad: number = 25;
        let esActivo: boolean = true;
        let lista: number[] = [1, 2, 3];

Esto permite que el compilador de TypeScript detecte errores de tipo antes de ejecutar el código.

Clases, Métodos, Interfaces, y Types

TypeScript ofrece una sintaxis de clase similar a la de otros lenguajes orientados a objetos. Además, introduce interfaces y tipos personalizados:

    
        // CLASES: Son estructuras definidas que sirven como modelos para crear objetos
        
        class Animal{
            nombre: string;


            constructor (nombre: string){
            this.nombre = nombre;
        }


            mover(distancia: number): void {
            console.log(`${this.nombre} se movió x metros.`);
        }
        }
        

        //  MÉTODOS: Es un bloque de código designado para desarrollar una tarea específica.
       
        function sumar(a: number, b: number): number {
            return a + b;
        }



        //  INTERFAZ: Las interfaces nos permiten definir la estructura de un 
        objeto, sus propiedades y métodos pero sin implementar funcionalidades.

        interface Persona {
            nombre: string;
            edad: number;
        }
       
        let juan: Persona = {
            nombre: "Juan",
            edad: 25
        };


        //TYPES: Ayudan a describir el tipo de datos con el que estamos trabajando
        type ID = number | string;


        let usuarioId: ID;


        usuarioId = 123;      // válido
        usuarioId = "abc123"; // también válido

Ciclos y Condicionales

TypeScript soporta las mismas estructuras de control que JavaScript, como if, for, while, etc.:

    
        //IF
   
        let numero: number = 5;
        if (true)
        {
        console.log('This will always executed.');
        }
   
        if (false) {
        console.log('This will never executed.');
        }   

        //SWITCH
    
        switch(expression) {
         case constant-expression1: {
             //statements;
             break;
        }
        case constant_expression2: {
             //statements;
             break;
        }
         default: {
              //statements;
            break;
        }
        }


        //FOR  

        for (let i = 0; i < 3; i++) {
            console.log ("Block statement execution no." + i);
        }


        //WHILE

        let i: number = 2;


        while (i < 4) {
        console.log( "Block statement execution no." + i )
        i++;
        }

Arrays, Listas, y Funciones

TypeScript maneja arrays y listas de manera similar a JavaScript, pero con la posibilidad de especificar el tipo de los elementos:

    
    let listaDeNumeros: number[] = [1, 2, 3, 4, 5];
    
    function sumar(a: number, b: number): number {
        return a + b;
    }
    
    let resultado = sumar(5, 10);
    console.log(`Resultado: ${resultado}`);

Angular

¿Qué es Angular?

Angular es un framework de desarrollo de aplicaciones web de código abierto, mantenido por Google. Está basado en TypeScript y está diseñado para facilitar la creación de aplicaciones web modernas, escalables y de alto rendimiento.

Características Importantes de Angular

  • Uso de DOM regular: Angular hace uso de DOM regular, lo que permite una mejor organización conforme avanza el desarrollo web.

  • Enlace de datos o Data Binding: Angular ofrece un mecanismo de enlace de datos bidireccional, lo que permite sincronizar automáticamente la interfaz de usuario con el modelo de datos.

  • Compatibilidad móvil y de escritorio: AAngular funciona tanto para el desarrollo de aplicaciones móviles como de escritorio. Esto también significa que puede ejecutarse en la mayoría de navegadores web.

  • Velocidad y rendimiento: Angular cuenta con código de generación que permite convertir tus plantillas en códigos altamente optimizados. Esto te ofrece todos los beneficios del código escrito a mano con la productividad de un marco.

  • Productividad: Angular permite la creación rápida de vistas de interfaz de usuarios con una sintaxis de plantilla muy sencilla y eficaz.

  • Enlace bidireccional de datos: Angular enlaza JavaScript y HTML, y una de sus principales ventajas es que el código de ambos está sincronizado, lo que ahorra mucho tiempo para los desarrolladores web.

  • Directivas: Existen diferentes tipos de directivas muy útiles que puedes usar para diferentes acciones como vincular el contenido de un elemento HTML a los datos de aplicación, o especificar que el contenido de un texto debe reemplazarse con una plantilla.

  • Pruebas: Angular hacer uso del framework de prueba llamado Jasmine, el cual proporciona diversas funcionalidades para escribir diferentes tipos de casos de prueba.

¿Cómo se Configura Angular?

Para configurar un proyecto en Angular, se utiliza Angular CLI (Command Line Interface). Aquí tienes los pasos básicos para crear y configurar una nueva aplicación Angular:

        
        // Instalar Angular CLI globalmente
        npm install -g @angular/cli
        
        // Crear un nuevo proyecto Angular
        ng new mi-app-angular
        
        // Navegar al directorio del proyecto
        cd mi-app-angular
        
        // Iniciar el servidor de desarrollo
        ng serve

Una vez que el servidor está en marcha, puedes acceder a tu aplicación Angular usando tu navegador, el cual normalmente se inicia en este puerto: http://localhost:4200.

Signal

En Angular, un Signal es un nuevo concepto introducido para facilitar la reactividad y el manejo de estado. Los Signals son flujos de datos reactivos que se utilizan para manejar la comunicación entre componentes o servicios, permitiendo que los cambios en el estado se propaguen automáticamente a través de la aplicación.


// Ejemplo de uso de Signal en Angular
import { Signal } from '@angular/core';

export class MiComponente {
    private _contador = Signal(0);

    incrementar() {
        this._contador.value++;
    }

    obtenerValor() {
        return this._contador.value;
    }
}
